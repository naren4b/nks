It should contain a property:

Without the property, analytics will not provide useful tracking.

## 8. Modernize and secure the GitHub Actions workflow

Your current deployment workflow uses:

* `actions/checkout@v3`
* `actions/setup-python@v4`
* unpinned Python package versions
* broad repository-level `contents: write`
* `mkdocs gh-deploy --force`

([GitHub][4])

Upgrade the actions and add dependency control:

```yaml
name: Deploy documentation

on:
  push:
    branches:
      - main
  pull_request:

permissions:
  contents: read

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: "3.12"
          cache: pip

      - name: Install dependencies
        run: pip install -r requirements.txt

      - name: Validate documentation
        run: mkdocs build --strict
```

For deployment, either use the official GitHub Pages workflow with `pages: write` and `id-token: write`, or keep `gh-deploy` in a separate job that only runs on `main`.

Create a `requirements.txt`:

```text
mkdocs-material==9.6.14
pillow==11.2.1
cairosvg==2.7.1
```

Use the latest compatible versions after testing; the example versions should not be copied blindly without validation.

Most importantly, add:

```bash
mkdocs build --strict
```

This will detect broken links, invalid navigation references and other documentation problems before deployment.
